# Sexta-feira, 05 de março, 2021

### 16:10 UTC-04

Hoje descobri que não se deve mais utilizar o nome *master* para a branch padrão
de um repositório git, mas sim, *main*. Eu confesso que não entendi as razões de
promover essa mudança que somente causa transtorno àqueles que utilizam a ferramenta.
Se esta palavra tem mesmo uma conotação assim tão ruim, então porquê perceberam
isso após décadas de uso? O uso desta palavra, por si só, realmente causa algum
tipo de ofensa em alguém? Se alguém se ofende com o nome de uma simples *branch*,
o que acontecerá no inevitável momento de um feeedback negativo? Não me importo
com o nome em si, só defendo que uma mudança como essa deveria ser feita por motivos
estritamente técnicos.
